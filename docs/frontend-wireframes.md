# 前端结构与样式草图

## 1. 页面目标

前端不是普通围棋对弈页，而是“对弈 + 分析 + 教学”的工作台。

因此页面不应该把棋盘单独做成主角，把分析塞到角落里；更合理的是：

- 棋盘负责操作
- 右侧分析区负责理解
- 下方历史区负责回看

## 2. 主页面信息架构

建议 MVP 只做一个主工作台页面：

- 左侧：棋盘区
- 右侧：分析与教学区
- 底部：着手历史和变化切换区
- 顶部：对局控制条

## 4. 移动端草图

移动端不要试图把桌面布局硬压缩。

建议顺序：

1. 顶部控制栏
2. 棋盘
3. 核心分析卡片
4. 推荐点与变化 tab
5. 历史面板折叠区

## 5. 功能摆放原则

### 顶部控制条

放这些：

- 新对局
- 执黑/执白状态
- 分析当前局面
- AI 应手
- 局部聚焦
- 终局复盘

不要放太多设置项。MVP 阶段把“高级设置”收进抽屉。

### 棋盘区

棋盘区需要承担：

- 落子交互
- 推荐点高亮
- 最后一手标记
- 聚焦区域框选
- 全盘 / 局部视图切换

建议棋盘左下角附一个小型状态浮层，显示：

- 当前行棋方
- 当前模式：全盘 / 局部
- 最近一次分析时间

### 右侧分析区

右侧区不要做成“很多小卡片堆满”。

建议固定为四段：

1. 局面总览
2. 教学摘要
3. 候选点列表
4. 细节 tab

这样用户阅读路径清楚：

- 先知道局面如何
- 再知道为什么
- 再看推荐手
- 最后看变化细节

### 底部历史区

底部建议用 tab，不建议同时展开很多面板。

tab 顺序建议：

- 着手历史
- 分析历史
- 教学说明
- 终局总结

## 6. 视觉方向

我建议前端不要做成传统“木纹棋盘 + 论坛风格面板”，也不要做成过度花哨的大厂运营页。

更适合的是：

- 整体基调：安静、专业、偏教学工作台
- 主色：暖白、墨黑、木色、少量青绿色强调
- 背景：浅米白或微暖灰，不要纯白刺眼
- 强调色：分析按钮和推荐点可用青绿色，不建议用紫色

建议的视觉语义：

- 棋盘区最重
- 教学摘要次重
- 历史区最轻

## 7. 组件优先级

### P0

- 棋盘组件
- 对局控制条
- 分析面板
- 推荐点列表
- 着手历史

### P1

- ownership 热力层
- 局部框选浮层
- 分析历史切换
- 终局复盘卡片

### P2

- 动画回放
- 题目模式入口
- 用户水平切换说明

## 8. 交互细节建议

### 点击“分析当前局面”

页面应：

- 显示加载状态
- 保持棋盘可见，不要弹全屏遮罩
- 分析完成后更新右侧面板
- 同时把结果写进底部“分析历史”

### 点击推荐点

页面应：

- 在棋盘上高亮该点
- 显示对应主变化
- 在教学区切换到该点的说明

### 切换局部聚焦

页面应：

- 保留全盘真实状态
- 只是改变当前主观察视口
- 明确显示“当前为局部聚焦模式”

## 9. 我建议的下一步

如果要正式开始前端，我建议下一步不是直接写页面，而是先产出：

1. 页面组件树
2. 前端状态树
3. 棋盘组件交互清单

这三项确定后，再开始写 Vue 或 React 页面，返工会少很多。
